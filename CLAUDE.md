<!-- K-433: /printing-press-import go.mod rewrite silently fails on CRLF line endings -->
- When writing `perl -pi -e` substitutions over text whose line endings you don't own (downloaded library tarballs, cross-platform staging, generator output), treat `\$` end-anchors as unsafe — on CRLF lines `$` lands between `\r` and `\n`, so a pattern like `s|^module \Q...\E\$|...|` silently misses. Bound with `\s+` (`s|^module\s+\Q...\E|...|`) or strip `\r` first (`perl -pi -e "s/\r\$//"`). After every `go.mod` module-path rewrite, assert with `grep -Eq "^module[[:space:]]+<expected>"` and exit non-zero on failure.
<!-- K-434: printing-press publish validate blocks on missing tools-manifest.json for pre-4.x CLIs -->
- Do NOT run `/printing-press-publish` on Windows for any CLI generated by press ≤3.x (or any CLI whose root is missing `tools-manifest.json`). `publish validate` fails the `manifest` check, `publish package` runs validate first and refuses, and there is no Windows path forward — `regen-merge` is macOS/Linux-only and `publish validate` has no flag to ignore a single check. Before suggesting publish, inspect the CLI's `.printing-press.json` for `printing_press_version` and check for `tools-manifest.json` at the CLI root; if missing or `< 4.0.0`, treat the CLI as publish-incompatible on Windows and route the retrofit to macOS/Linux (or block on an upstream press fix).
